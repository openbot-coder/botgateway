### 问题1：脚本报错后跳过流程

**规范要求**（SKILL.md 第 337 行）：

> "遇到问题及时反馈：不要自己埋头解决，第一时间向用户报告问题"

**犯的错误**：

- 在验证失败时继续执行，没有及时向用户报告问题
- 没有按照规范的调试流程（根因调查 4 阶段）来修复问题

**正确做法**：

1. 发现问题 → 立即停止并向用户报告
2. 进行根因调查（4 个阶段）
3. 修复 → 重新验证
4. **绝不绕过问题继续执行**

### 问题2：没有遵循 TDD 流程

**规范要求**（SKILL.md 第 289-308 行）：

> "**TDD 铁律：没有失败的测试就不能写生产代码**"
> "**红-绿-重构循环：红（写失败测试）→ 绿（最少代码通过）→ 重构（清理）**"
> "**单元测试覆盖率目标：100%**"

**犯的错误**：

- 先写代码，后来才补测试（违反 TDD 铁律）
- 没有在每个功能点遵循红-绿-重构循环
- 测试覆盖率未达到 100%

**正确做法（TDD 流程）**：

```
1. [红] 编写一个失败的测试（验证功能不存在）
2. [绿] 编写最少量代码使测试通过
3. [重构] 清理代码，保持测试通过
4. 重复以上步骤直到所有功能点完成
```

**TDD 测试三维度覆盖**：

- **正例**：正常输入 → 正常输出
- **反例**：无效输入 → 正确错误响应
- **边界值**：最小值、零值、空值、最大值

### 问题3：归档执行不到位

**规范要求**（SKILL.md 第 383-413 行）：

> "运行 `python scripts/archive.py --name <变更名称>`"
> 脚本将：
> - 提交 Git
> - 推送到远程仓库
> - 询问是否打标签
> - 将变更目录移动到 `archive/`"

**犯的错误**：

- 没有使用 `archive.py` 脚本
- 遗漏了创建 PR 的步骤（应该在归档前完成）
- 没有按照标签格式打标签（`v<version>/<变更名称>`）
- 变更目录没有移动到 `archive/`

**正确归档流程**：

```bash
python scripts/archive.py --name <变更名称>
# 自动完成：
# 1. Git 提交
# 2. 推送到远程仓库
# 3. 询问是否打标签（格式：v0.2.0/create-v0-2-0）
# 4. 移动到 archive/
```

**标签格式**：

- `v<version>/<变更名称>`（如 `v1.0/add-dark-mode`）
